# Data Structure Documentation

## Overview

The HSK lesson site uses a **normalized data structure** to avoid duplication and maintain consistency across vocabulary, characters, and lessons.

## Data Files

### 1. `characters.js` - Character Dictionary
**Purpose:** Single source of truth for individual Chinese character data.

**Structure:**
```javascript
window.HSK_CHARACTERS = {
  "你": { p: "nǐ", e: "you" },
  "好": { p: "hǎo", e: "good" }
  // ... more characters
};
```

**Usage:** Referenced by vocabulary breakdown and stroke order display.

---

### 2. `vocabulary.js` - Word Decomposition
**Purpose:** Centralized word deconstruction data (parts + literal meanings).

**Structure:**
```javascript
window.HSK_VOCABULARY = {
  "你好": {
    parts: ["你", "好"],
    literal: "you + good = hello"
  }
  // ... more words
};
```

**Usage:** Modal fetches breakdown data using `HSK_VOCAB_UTILS.getBreakdown(word)`.

---

### 3. `constructs.js` - Backwards Compatibility
**Purpose:** Generated from `characters.js` for compatibility.

**Structure:**
```javascript
window.HSK_CONSTRUCTS = {
  "你": { c: "你", p: "nǐ", e: "you" }
};
```

**Status:** Auto-generated by `characters.js`. Can be deprecated if not needed.

---

### 4. `lessons/*.js` - Lesson Content
**Purpose:** Lesson-specific content (vocab list, grammar, sentences, dialogue).

**Structure:**
```javascript
window.HSK_LESSONS_BY_SLUG["lesson-1"] = {
  vocab: [
    { c: "你好", p: "nǐ hǎo", e: "hello" }
    // NO parts/literal here - fetched from vocabulary.js
  ],
  grammar: [...],
  sentences: [...],
  dialogue: [...]
};
```

**Key principle:** Lessons store only `{c, p, e}`. Breakdown data lives in `vocabulary.js`.

---

### 5. `dictionary.js` - Dictionary Data
**Purpose:** Complete HSK 1 word list with categories.

**Structure:**
```javascript
window.HSK_DICTIONARY = {
  words: [
    { c: "你好", p: "nǐ hǎo", e: "hello", cat: "greeting" }
  ],
  categories: {
    "greeting": "Greetings · 问候"
  }
};
```

---

## Data Flow

```
User clicks vocab word
    ↓
Modal opens with {c, p, e}
    ↓
Breakdown: HSK_VOCABULARY[word] → parts, literal
    ↓
Character info: HSK_CHARACTERS[char] → p, e
    ↓
Stroke order: HanziWriter API → stroke animation
```

## Adding New Content

### Add a new lesson vocabulary word:
1. Add to `lessons/lesson-X.js`: `{ c: "新词", p: "xīncí", e: "new word" }`
2. If multi-character, add to `vocabulary.js`: `"新词": { parts: ["新", "词"], literal: "..." }`
3. If new characters, add to `characters.js`: `"新": { p: "xīn", e: "new" }`

### Add character data:
1. Update `characters.js` with `{ p: "pinyin", e: "meaning" }`
2. `constructs.js` updates automatically via backwards compatibility code

## Migration Notes

- Old lesson files had `parts` and `literal` inline with vocab items
- These have been moved to `vocabulary.js` for normalization
- Lessons now contain only basic vocab data `{c, p, e}`
- Modal fetches breakdown dynamically from centralized sources

## Future Improvements

1. Consider removing `constructs.js` entirely if no dependencies exist
2. Generate `characters.js` from `vocabulary.js` + lesson vocab automatically
3. Add validation script to ensure data consistency
